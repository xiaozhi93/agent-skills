# Phase 1: 访谈与澄清

**目标：** 产出「文章意图简报」——对齐写什么、给谁看、怎么写、不写什么。

## 输入

- 当前会话的完整对话记录
- 用户补充说明（如有）

## 步骤

### 1.1 扫描对话，识别素材类型

从对话中提取并分类（内部工作，不向用户展示原始分类表）：

| 类型 | 识别标志 |
|------|----------|
| 核心洞见 | 用户或 Agent 提出的框架、判断、方法论 |
| 论证过程 | 推理链、正反讨论、证据 |
| 案例/实例 | 具体场景、数据、代码片段 |
| 试错/闲聊 | 跑题、重复、语气词、操作失败记录 |
| 敏感信息 | 人名、内部项目名、未公开数据 |

### 1.2 结构化访谈（一次一问）

按顺序向用户提问，**每次只问一个问题**，等回答后再问下一个：

1. **主题**：这篇文章的核心主题是什么？（用一句话）
2. **受众**：谁读？（如：同事、公众、初学者、领域专家）
3. **目的**：读完后希望他们做什么/理解什么？
4. **调性**：正式报告 / 技术博客 / 轻松随笔 / 教程？
5. **范围**：对话中哪些部分必须保留？哪些明确排除？
6. **脱敏**：有没有需要匿名化或删除的信息？
7. **格式**：Markdown？公众号？字数限制？需要标题/摘要/关键词吗？

若用户已在初始请求中回答了部分问题，跳过已答项。

### 1.3 输出文章意图简报

```markdown
## 文章意图简报

**主题：** …
**受众：** …
**目的：** …
**调性：** …
**包含范围：** …
**排除范围：** …
**脱敏要求：** …
**格式要求：** …

### 对话精华预览（3-5 条）
1. …
2. …
…

### 待确认
- [ ] 以上理解是否准确？
- [ ] 精华预览是否覆盖了你想保留的核心？
```

## 门禁

**必须获得用户明确确认**（如「可以」「确认」「继续」）后才能进入 Phase 2。

用户要求修改 → 更新简报 → 重新确认。

## 常见失误

| 失误 | 对策 |
|------|------|
| 一次抛出 7 个问题 | 严格遵守一次一问 |
| 替用户决定受众和调性 | 必须询问 |
| 遗漏脱敏 | 第 6 问不可跳过 |
