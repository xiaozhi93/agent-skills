# Phase 3: 生成内容

**目标：** 按已确认的大纲和素材映射，写出文章初稿。

## 输入

- Phase 1 文章意图简报
- Phase 2 已确认的大纲 + 素材映射表
- 完整对话记录

## 写作原则

### 3.1 对话 → 文章的转换规则

| 对话特征 | 文章处理 |
|----------|----------|
| 口语、重复、语气词 | 删除或凝练 |
| 高质量原话/框架 | 保留或紧密 paraphrase，可加引用块 |
| 来回讨论 | 合并为一段连贯论述 |
| 试错过程 | 只保留结论和关键转折 |
| 代码/命令 | 整理为规范代码块，加必要注释 |
| 列表/表格 | 保留并格式化 |

### 3.2 叙事要求

- **有钩子**：引言第一段抓住读者
- **有主线**：每节服务于 Phase 2 的主论点
- **有过渡**：章节间用过渡句连接，禁止突兀跳跃
- **有收尾**：结论回扣主题，给出明确 takeaway

### 3.3 风格约束

严格遵守 Phase 1 确认的：

- 受众水平（术语解释程度）
- 调性（正式/轻松）
- 字数范围
- 脱敏要求

### 3.4 输出格式

```markdown
---
title: …
summary: …
tags: […]
source: conversation
---

# 标题

> 摘要（如需要）

正文…

---

*本文由对话整理生成。*
```

根据 Phase 1 格式要求调整 frontmatter 和排版（如公众号不含 YAML frontmatter）。

## 禁止事项

- **禁止**按对话时间顺序逐轮复述
- **禁止**引入对话中未出现的观点或数据
- **禁止**保留 Phase 1 明确排除的内容
- **禁止**保留敏感信息（人名、内部代号等，除非用户明确保留）

## 产出

**文章初稿** — 完成后自动进入 Phase 4，不向用户宣告「已完成」。

## 常见失误

| 失误 | 对策 |
|------|------|
| 写成对话实录 | 对照映射表，按章节重组 |
| 金句被过度意译 | 映射表标记「保留核心」的用引用块 |
| 篇幅失控 | 对照预计字数，超 20% 需精简 |
