# Phase 2: 编排文章结构

**目标：** 将杂乱对话重组为可读的文章骨架，并建立「大纲 ↔ 对话素材」映射。

## 输入

- Phase 1 已确认的文章意图简报
- 完整对话记录

## 步骤

### 2.1 提炼核心论点

从对话中归纳 **1 个主论点 + 2-4 个支撑论点**，不按时间线排列。

### 2.2 选择文章结构模式

根据主题和受众选择最合适的模式：

| 模式 | 适用场景 |
|------|----------|
| 问题 → 分析 → 结论 | 思辨、决策、方法论 |
| 背景 → 实践 → 复盘 | 技术实践、项目经验 |
| 观点 → 论证 → 反驳 → 回应 | 有争议的话题 |
| 概念 → 原理 → 示例 → 应用 | 教程、科普 |
| 清单体 | 多要点并列、工具对比 |

### 2.3 生成大纲

```markdown
## 文章大纲

**标题（草案）：** …
**摘要（1-2 句）：** …

### 结构

1. **引言** — 钩子 + 读者收益
2. **…** — …
3. **…** — …
4. **结论** — 核心 takeaway

**预计字数：** …
```

### 2.4 生成素材映射表

每个大纲节点必须映射到对话来源，防止写作时遗漏或编造：

```markdown
## 素材映射表

| 章节 | 对话来源要点 | 保留/改写/排除 | 备注 |
|------|-------------|---------------|------|
| 引言 | 对话第…轮：… | 改写 | 去掉口语 |
| §2 … | 对话第…轮：… | 保留核心 | 金句直引 |
| … | … | … | … |
```

**规则：**

- 每个章节至少 1 条映射
- 标记为「排除」的素材不得进入 Phase 3
- 对话中的高质量原话标记「保留核心」，Phase 3 优先直引或紧密 paraphrase

### 2.5 输出并等待确认

向用户展示 **大纲 + 素材映射表**，询问：

- 章节顺序是否合理？
- 有无遗漏或多余章节？
- 标题方向是否满意？

## 门禁

**必须获得用户明确确认**后才能进入 Phase 3。

结构调整 → 更新大纲和映射表 → 重新确认。

## 常见失误

| 失误 | 对策 |
|------|------|
| 按对话轮次排章节 | 按论点逻辑排，非时间线 |
| 大纲过细（写到段落级） | 精确到章节 + 要点即可 |
| 无素材映射 | 映射表是必填项 |
| 把所有对话内容塞进大纲 | 尊重 Phase 1 排除范围 |
