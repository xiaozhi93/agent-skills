# Phase 4: 验证结构与内容

**目标：** 系统性检查文章质量，修复问题后交付终稿。

## 输入

- Phase 1 文章意图简报
- Phase 2 大纲 + 素材映射表
- Phase 3 文章初稿

## 验证清单

逐项检查，**每项标记 ✅ 或 ❌**：

### A. 结构验证

| # | 检查项 |
|---|--------|
| A1 | 标题准确反映主题 |
| A2 | 引言有钩子且说明读者收益 |
| A3 | 章节顺序符合 Phase 2 大纲 |
| A4 | 每节有明确论点，无跑题段落 |
| A5 | 章节间过渡自然 |
| A6 | 结论回扣主题并给出 takeaway |
| A7 | 无按对话时间线堆砌的段落 |

### B. 内容验证

| # | 检查项 |
|---|--------|
| B1 | 核心洞见均已覆盖（对照素材映射表） |
| B2 | 无对话中未出现的事实或观点 |
| B3 | 无 Phase 1 排除的内容 |
| B4 | 脱敏要求已执行 |
| B5 | 代码/数据/引用准确（对照原对话） |
| B6 | 调性与受众匹配 |
| B7 | 字数在约定范围内（±20%） |

### C. 可共享性验证

| # | 检查项 |
|---|--------|
| C1 | 非参与者能读懂，无需对话上下文 |
| C2 | 无内部缩写/黑话未解释 |
| C3 | 无隐私或敏感信息泄露 |
| C4 | 文章可独立发布，不依赖会话链接 |

## 输出验证报告

```markdown
## 验证报告

### 结构验证
- A1 ✅ …
- A2 ❌ 引言缺少钩子 → 已修复
…

### 内容验证
…

### 可共享性验证
…

### 处置
- 回流 Phase 2：…（如有结构问题）
- 回流 Phase 3：…（如有内容问题）
- 通过：交付终稿
```

## 回流规则

| 问题类型 | 回流目标 |
|----------|----------|
| 章节缺失/顺序错误/叙事逻辑断裂 | Phase 2 调整大纲 |
| 内容遗漏/事实错误/风格偏差/脱敏遗漏 | Phase 3 修改初稿 |
| 仅个别措辞问题 | 直接在终稿中修复，标注在报告中 |

**最多回流 2 轮。** 第 3 轮仍有关键项 ❌ 时，向用户说明未通过项并请求人工决策。

## 门禁

全部检查项 ✅ 后，输出 **终稿** 并简短说明：

- 文章主题与受众
- 相比原始对话做了哪些取舍
- 有无需要用户最终确认的判断

## 常见失误

| 失误 | 对策 |
|------|------|
| 跳过清单直接说「写好了」 | 必须逐项勾选 |
| 全部标 ✅ 但有问题 | 对照映射表和简报逐条核实 |
| 验证后大改但不更新报告 | 修复须记录在报告中 |
