# Phase 3: 产出交接文档

**目标：** 基于蒸馏报告，产出结构化交接文档 + Canvas 架构图，供 Phase 4 实现使用。

**前置：** Phase 2 蒸馏报告已确认。

## 步骤

### 3.1 生成 Canvas 架构图

使用 `canvas` 技能，呈现：

- 模块/分层关系（源仓视角）
- 主流程（序列或数据流）
- 源 → 目标映射预览（如已知目标仓结构）

Canvas 是**必交付物**，不可仅用长文替代。

### 3.2 填写交接模板

复制 [templates/handoff-template.md](../templates/handoff-template.md)，替换全部 `{...}` 占位符。

**质量要求：**

- **自包含**：Phase 4 的 Agent 无需翻阅 Phase 2 对话即可开工
- **可执行**：文件路径、步骤、验收标准必须具体
- **可判定**：每条 P0 验收标准能明确判断完成与否
- **适配导向**：明确「源仓做法 → 目标仓等价做法」，禁止「复制某文件」

### 3.3 跨语言/跨框架专节

若 Phase 1 标记跨语言或跨框架，必须填写模板中的「适配说明」：

- API/类型映射表
- 目标语言惯用模式替代方案
- 无法直接对等的行为及取舍

### 3.4 向用户交付

1. Canvas 链接或截图说明
2. 完整交接文档（Markdown）
3. **可选**：用户要求时写入 `docs/handoff/YYYY-MM-DD-{feature-slug}.md`

## 输出格式

```markdown
## 交接文档已就绪

**功能：** …
**Canvas：** {链接或说明}

### 摘要
- 实现步骤数：N
- P0 验收标准：N 条
- 跨语言适配：是 / 否

{完整交接文档正文，或指向保存路径}

### 待确认
- [ ] 交接文档是否批准？批准后开始 Phase 4 实现。
```

## 门禁

用户明确批准交接文档后 → Phase 4。

**禁止：**

- 交接文档含 `{待填写}` 等空占位符
- 在 Phase 3 同时修改目标仓库代码
- 跳过 Canvas
