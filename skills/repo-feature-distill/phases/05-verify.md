# Phase 5: 验证

**目标：** 确认目标仓库实现与源仓功能对等，产出验证报告。

**前置：** Phase 4 实现已完成展示。

## 验证清单

### 功能对等

```
- [ ] 每条 Phase 1 P0 验收标准已验证
- [ ] 主流程端到端可走通
- [ ] 边界情况与源仓行为一致（或偏差已在报告中说明）
```

### 代码质量

```
- [ ] 遵循目标仓命名、目录、风格约定
- [ ] 无 linter 错误引入（如项目有检查）
- [ ] 无源仓残留（错误 import 路径、未适配 API 等）
```

### 交接文档符合度

```
- [ ] 实现清单各项已覆盖
- [ ] 偏差项已记录原因
- [ ] 开放问题已关闭或标注后续待办
```

## 验证方式

按交接文档「测试与验收」章节执行：

- 手动验证场景
- 自动化测试命令（如适用）
- 必要时回读源仓关键路径做行为对比

### 处置

## 门禁

全部 P0 验收通过 → 任务完成。

**未通过时：**

- 实现问题 → 回流 Phase 4
- 理解/交接遗漏 → 回流 Phase 3 或 Phase 2
