# Phase 4: 目标仓库实现

**目标：** 在目标仓库按交接文档适配实现功能——行为对等，代码遵循目标仓约定。

**前置：** Phase 3 交接文档已批准；Agent 位于目标仓库工作区。

## 实施原则

1. **先读再写**：阅读交接文档「目标仓现有模式」章节，浏览参考文件
2. **适配不复制**：理解源仓意图，用目标仓惯用方式表达；禁止整段粘贴源仓代码
3. **分步验证**：每完成一个逻辑块运行验证，不累积大批量未测改动
4. **最小依赖**：不引入交接文档未讨论的新依赖
5. **影响评估**：改动前可用 `gitnexus-impact-analysis` 评估（GitNexus 已索引时）

## 实施顺序

```
1. 阅读交接文档 + 目标仓核心参考文件
2. 按「实现清单」逐步执行（从底层/依赖少项开始）
3. 每步完成后对照 P0 验收标准自检
4. 展示变更摘要（文件列表 + 关键决策）
```

## 跨语言/跨框架注意

- 参照交接文档「适配说明」，不做 literal port
- 类型系统、异步模型、错误处理方式按目标语言重写
- 接口行为对等优先于实现细节一致

## 门禁

实现展示后 → Phase 5 验证。

**禁止：**

- 偏离交接文档 In scope 擅自扩展
- 1:1 复制源仓文件结构到风格迥异的目标仓
- 跳过验证步骤直接宣称完成
