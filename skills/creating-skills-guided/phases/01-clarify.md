# Phase 1: 访谈与澄清

**目标：** 产出「技能意图简报」——对齐做什么、何时触发、放哪里、不做什么。

**参考：** `create-skill` Discovery 阶段，但增加用户确认门禁。

## 步骤

### 1.1 一次一问

按顺序提问，**每次只问一个问题**：

1. **目的**：这个技能解决什么具体问题？（一句话）
2. **触发**：什么情况下 Agent 应自动/显式启用？（症状、关键词）
3. **范围**：包含什么？明确不做什么？
4. **存放**：个人 `~/.cursor/skills/` 还是项目 `.cursor/skills/`？
5. **类型**：Technique / Pattern / Reference / 多阶段工作流？
6. **约束**：有无篇幅、格式、门禁、脚本依赖？
7. **参考**：有无现有技能或对话经验可对齐？

已答项跳过。

### 1.2 输出技能意图简报

```markdown
## 技能意图简报

**工作名（草案）：** skill-name-kebab-case
**目的：** …
**触发场景：** …
**包含范围：** …
**排除范围：** …
**存放路径：** ~/.cursor/skills/… 或 .cursor/skills/…
**技能类型：** …
**特殊约束：** …

### 待确认
- [ ] 以上是否准确？
```

## 门禁

用户明确确认（「可以」「确认」「继续」）后 → Phase 2。
