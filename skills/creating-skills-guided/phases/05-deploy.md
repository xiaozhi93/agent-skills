# Phase 5: 产出与部署

**目标：** 将验证通过的技能写入目标路径，并告知用户使用方式。

## 步骤

### 5.1 写入文件

- 确认目标路径与 Phase 1 一致
- 写入 SKILL.md 及所有附属文件
- **禁止**写入 `~/.cursor/skills-cursor/`

### 5.2 可选：更新分类索引

若用户维护 `SKILL-TAXONOMY.md`，追加一行归类记录。

### 5.3 交付报告

```markdown
## 技能已部署

**名称：** skill-name
**路径：** /absolute/path/to/skill-name/
**触发：** （复述 description 触发条件）

### 如何使用
- 显式调用：`/skill-name` 或对话中提及触发词
- 附属文件：…

### 后续建议
- 运行 writing-skills 完整 subagent 压测（若 Phase 4 仅草案）
- 在真实任务中用 1–2 次，按 REFACTOR 堵新漏洞
```

### 5.4 会话结束规则

**当前技能部署并报告完成前，不得开始创建下一个技能。**

## 常见失误

| 失误 | 对策 |
|------|------|
| 验证未通过就写入 | Phase 4 门禁 |
| 路径与简报不一致 | 对照 Phase 1 |
| 部署后未告诉用户如何触发 | 交付报告必含触发说明 |
