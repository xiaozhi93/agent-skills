# Phase 5: 验证

**目标：** 产出验证报告，证明收敛满足 Phase 1 成功标准与 Phase 3 统一标准。

**必须**遵循 `verification-before-completion`。

## 验证清单

```
- [ ] 单元/集成测试通过（贴命令与输出摘要）
- [ ] 构建通过（若适用）
- [ ] 各原实现入口行为抽样一致（或 documented 差异）
- [ ] 冗余代码已删除或 deprecated 标记正确
- [ ] 无 Phase 3 未批准的额外行为变更
- [ ] 跨仓场景：各仓独立验证通过
```

## 输出验证报告

```markdown
## 验证报告

**业务域：** …
**结论：** 通过 / 不通过

### 证据

| 检查项 | 命令/方法 | 结果 |
|--------|-----------|------|
| … | … | … |

### 遗留项（若有）

…
```

## 门禁

**通过** → Phase 6。  
**不通过** → 回流 Phase 4，修复后重新验证。
