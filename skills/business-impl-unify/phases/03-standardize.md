# Phase 3: 对比定标

**目标：** 产出「差异报告 + 统一标准提案」——对比各套实现，提出可执行的收敛方案。

**前置：** Phase 2 实现清单完整。

## 对比维度

| 维度 | 对比内容 |
|------|----------|
| 行为 | 输入/输出、边界条件、错误语义 |
| 结构 | 分层、模块边界、命名 |
| 依赖 | 框架、第三方库、配置 |
| 测试 | 覆盖范围、测试类型 |
| 运维 | 日志、监控、特性开关 |

使用模板：[templates/divergence-report.md](../templates/divergence-report.md)

## 统一标准提案

必须回答：

1. **保留什么**：共享抽象、目录、公共模块路径
2. **废弃什么**：哪些实现/文件在收敛后删除或标记 deprecated
3. **迁移策略**：逐模块替换 /  strangler / 一次性切换
4. **适配说明**：跨仓或跨语言时的映射规则
5. **风险与回滚**：如何验证、如何回退

使用模板：[templates/standard-proposal.md](../templates/standard-proposal.md)

## 输出格式

先呈现差异报告，再呈现统一标准提案，分开两个章节，便于用户审阅。

## 门禁

**用户明确批准统一标准后** → Phase 4。

未批准前：**禁止**写收敛代码、禁止删除旧实现。
